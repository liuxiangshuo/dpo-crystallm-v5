# 改进方案：如何真正提升晶体结构生成的稳定率

## 核心发现：为什么 DPO 失败了，以及突破口在哪里

### 空间群是决定稳定性的关键

| 空间群 | 生成占比 | 稳定率 | 对总稳定率的贡献 |
|--------|---------|--------|----------------|
| **P2₁** | **26.3%** | **0.4%** | 0.11% |
| **Pnma** | 25.9% | 14.1% | 3.65% |
| **Cmcm** | 13.0% | **39.8%** | **5.17%** |
| Pna2₁ | 10.5% | 19.7% | 2.07% |
| Pc | 8.5% | 5.2% | 0.44% |
| P-1 | 6.9% | 9.2% | 0.63% |
| Cc | 4.9% | 0.0% | 0.00% |
| Imma | 3.9% | 0.1% | 0.00% |

**关键洞察**: 模型花 26.3% 的生成能力在 P2₁ 空间群上，但其中只有 0.4% 是稳定的。
如果能把这部分生成能力转移到 Cmcm（39.8% 稳定）和 Pna2₁（19.7% 稳定），
稳定率可以从 12% 提升到 **18-25%**。

### DPO 为什么不能做到这一点

DPO 在偏好对中确实有强烈的空间群信号（Pna2₁:Chosen 42% vs P2₁:Rejected 61%），
但训练后生成分布 **完全没变**（空间群比例差异 < 0.2%）。

原因：单一 prompt 下，15,000 个偏好对的梯度方向互相矛盾，pi_delta 训练至 0.05（≈随机）。

---

## 方案 A：SFT on Stable Structures（推荐，最高可行性）

### 原理

直接在 6,062 个稳定结构上做 Supervised Fine-Tuning。
不需要偏好对，不需要 rejected 样本，只学"什么是好的"。

### 为什么它能解决问题

1. **无梯度冲突**: 所有训练样本都朝同一个方向推
2. **自然偏移空间群分布**: 训练数据中 Cmcm=43%, Pnma=30%, 模型会学到偏好这些空间群
3. **直接优化**: 标准交叉熵损失，信号强且稳定
4. **理论保障**: 等价于 ReST/RAFT（Rejection Sampling Fine-Tuning），已在 LLM 对齐中验证有效

### 预期效果

| 场景 | 预期稳定率 | 相比 Baseline |
|------|-----------|-------------|
| 乐观（完全偏移） | ~25% | **+13%（2.1倍）** |
| 保守（50% 偏移） | ~19% | **+7%（1.6倍）** |
| 悲观（20% 偏移） | ~15% | +3% |

### 实现步骤

```
Step 1: 从 baseline 50K 中筛选 6,062 个稳定 CIF
Step 2: 将 CIF 转为 CrystaLLM 训练格式（prompt + completion）
Step 3: 用交叉熵损失微调 CrystaLLM（LR=1e-7, 2000-4000 步）
Step 4: 用微调后的模型生成 50K CIF
Step 5: 评估稳定率、多样性、新颖性
```

### 关键注意事项

- 用低学习率（1e-7）防止灾难性遗忘
- 可选 LoRA 微调减少参数量
- 监控生成多样性（空间群种类数、晶胞参数分布）
- 训练约 3-5 个 epoch（6062 × 3 ≈ 18,000 步）

### 代码改动量

需要编写 1 个新脚本（SFT 训练脚本），改动 driver 脚本。
可以复用现有的 `32_train_dpo_crystallm.py` 中的模型加载代码。
**预计 1-2 天完成实现 + 实验。**

## 方案 B：Iterative SFT（ReST-EM）

### 原理

SFT 的迭代版本，每轮用新模型生成更多稳定样本，再加入训练集。

### 流程

```
Round 0: SFT on 6,062 stable CIFs → Model_0
Round 1: Model_0 generates 10K CIFs → filter → ~2000 new stable → SFT on 8,062 → Model_1
Round 2: Model_1 generates 10K CIFs → filter → more stable → SFT → Model_2
...
```

### 优势

- 每轮增加更多多样的稳定样本
- 逐步收敛到高稳定率
- 文献支撑: ReST-EM (Google 2023), Self-Play (Meta 2024)

### 缺点

- 需要多轮生成（每轮 ~20h），总实验时间较长
- 可能过拟合到特定结构类型

---

## 方案 C：Intra-Space-Group DPO（让 DPO 真正有效）

### 原理

不再在不同空间群之间做偏好对比，而是在 **同一空间群内** 做微调。

### 关键改动

1. **构建同空间群偏好对**:
   - 在 Cmcm 内: 低能量 Cmcm vs 高能量 Cmcm
   - 在 Pnma 内: 低能量 Pnma vs 高能量 Pnma
2. **多 prompt**:
   - `data_Li4Fe4P4O16\n..._symmetry_space_group_name_H-M Cmcm`
   - `data_Li4Fe4P4O16\n..._symmetry_space_group_name_H-M Pnma`
3. **更小的 token 差异**: 同空间群内的结构差异主要在晶胞参数和原子坐标

### 优势

- 解决了单一 prompt 和梯度冲突问题
- DPO 信号更精细（学习结构参数而非空间群选择）
- 可以与方案 A 组合：先 SFT 偏移空间群分布，再用 DPO 细调参数

### 缺点

- 实现更复杂
- 同空间群内的能量差异较小，可能信号仍然偏弱
- 需要确认 CrystaLLM 支持 longer prompt

---

## 方案 D：SFT + DPO 两阶段（最佳长期方案）

### 流程

```
Stage 1: SFT on stable structures → 空间群分布偏移 → 稳定率 ~18-25%
Stage 2: 用 SFT 模型生成新数据 → 构建同空间群偏好对 → DPO 细调
         → 在每个空间群内优化结构参数 → 稳定率进一步提升
```

### 优势

- Stage 1 解决"粗粒度"问题（空间群选择）
- Stage 2 解决"细粒度"问题（结构参数优化）
- 两阶段都有明确的物理解释

---

## 推荐优先级

| 优先级 | 方案 | 预期效果 | 实现难度 | 实验时间 |
|--------|------|---------|---------|---------|
| **★★★** | **A: SFT on Stable** | 稳定率 15-25% | 低 | 2-3 天 |
| ★★☆ | B: Iterative SFT | 稳定率 20-30% | 中 | 1 周 |
| ★★☆ | C: Intra-SG DPO | 稳定率 13-18% | 中 | 3-4 天 |
| ★★★ | D: SFT + DPO | 稳定率 20-35% | 中高 | 1-2 周 |

**建议路线**: 先做方案 A（2-3 天出结果），如果稳定率显著提升，继续做方案 D 的 Stage 2。
