## 2. 训练动态分析

### 2.1 损失曲线收敛情况

| 实验 | 初始 Loss | 最终 Loss | 下降幅度 | Loss 类型 |
|------|----------|----------|---------|----------|
| exp_final_50k | 7.699 | 4.419 | -3.280 | sum log-prob |
| exp_ablation_dpo | 0.809 | 0.655 | -0.154 | per-token avg |
| exp_ablation_cdpo | 0.800 | 0.672 | -0.128 | per-token avg |
| exp_ablation_simpo | 1.517 | 1.294 | -0.222 | per-token avg |

**发现 1**: 新实验的损失下降非常平缓（仅 0.13-0.22），说明模型在训练中几乎没有学到有效的偏好信号。对比原始实验损失下降 3.28，差距极大——但这主要是因为 log-prob 计算方式不同（sum vs per-token average）。

### 2.2 策略偏移分析 (pi_delta)

pi_delta = log π(chosen) - log π(rejected)，正值表示策略模型更偏好 chosen。

| 实验 | 训练末期 pi_delta | pi_delta > 0 比例 | 标准差 |
|------|-------------------|-------------------|--------|
| exp_final_50k | 866.26 | 90% (45/50) | 1364.36 |
| exp_ablation_dpo | 0.050 | 54% (27/50) | 0.27 |
| exp_ablation_cdpo | 0.045 | 54% (27/50) | 0.26 |
| exp_ablation_simpo | 0.053 | 58% (29/50) | 0.27 |

**发现 2（核心问题）**: 新实验的 pi_delta 接近 0，且仅有 54% 的正值率（接近随机猜测的 50%）。这意味着**训练 8000 步后，模型基本没有学会区分 chosen 和 rejected**。

### 2.3 梯度范数

| 实验 | 早期梯度 | 后期梯度 | 下降比例 |
|------|---------|---------|---------|
| exp_final_50k | 476.6 | 183.4 | 2.6x |
| exp_ablation_dpo | 14.8 | 10.6 | 1.4x |
| exp_ablation_cdpo | 12.3 | 8.4 | 1.5x |
| exp_ablation_simpo | 16.1 | 13.2 | 1.2x |

**发现 3**: 新实验的梯度范数比原始实验小 30-50 倍，且下降幅度有限。模型权重几乎没有被有效更新。

## 3. 偏好对质量分析

### 3.1 基本统计

- **总对数**: 15,000 对
- **唯一 prompt**: **仅 1 个** (`data_Li4Fe4P4O16`)
- **唯一 chosen CIF**: 5,215 个
- **唯一 rejected CIF**: 5,201 个
- **Chosen/Rejected 重叠**: 0（没有同一个 CIF 同时出现在两边）

### 3.2 能量差分析

| 指标 | 值 |
|------|---|
| 均值差距 | 5.55 eV/atom |
| 中位差距 | 3.37 eV/atom |
| 最小差距 | 1.82 eV/atom |

**发现 4**: 能量差距非常大（全部 > 1.8 eV/atom），这看起来是好事（信号清晰），但实际上反映了 chosen 和 rejected 是**完全不同的晶体结构**，而非同一结构类型的微调。

### 3.3 CIF 内容差异分析

| 指标 | 值 |
|------|---|
| 共同前缀 | ~221 字符 (元素类型定义) |

**发现 5（最关键）**: Chosen 和 Rejected 从空间群名称开始就完全分叉。两者代表的是截然不同的晶体结构（不同空间群、不同晶胞参数、不同原子坐标）。DPO 不是在学"微调同一个结构使其更稳定"，而是在学"选择 Pna2₁ 空间群而非 P2₁ 空间群"。

## 5. 根本原因分析

### 根因 1: 单一 Prompt 下的 DPO 训练本质上无效

DPO 的核心机制是：给定 prompt x，学习 `π(y_chosen|x) > π(y_rejected|x)`。当所有 15,000 对共享同一个 prompt `data_Li4Fe4P4O16` 时：

- 模型需要在**同一个上下文**下同时提高 5,215 个 chosen 序列的概率、降低 5,201 个 rejected 序列的概率
- 但这些序列之间差异巨大（不同空间群、不同结构），彼此冲突
- DPO 梯度在这些冲突信号中相互抵消

### 根因 2: Token 级别的信号稀释

- CIF 序列长度 452-667 tokens
- 前 ~100 个 tokens（元素类型定义）在所有序列中完全相同
- Per-token average log-prob 将差异分散到整个序列，使有效信号被稀释 5-7 倍
- 最终 pi_delta ≈ 0.05，远小于有意义的训练信号

### 根因 3: 有效训练次数不足

- 15,000 对，8,000 步 → 每对平均只被看到 0.5 次（epoch < 1）
- 梯度累积 16 步 = 有效 batch size 16，但数据量不足以充分覆盖

### 根因 4: DPO 不适合"生成多样结构后择优"的任务

DPO 设计用于对齐——让模型学会在多种可能回答中偏好更好的那种。但对于晶体生成：
- 模型本身已经在以合理的概率生成稳定结构（12.12% baseline）
- Best-of-N 采样（N=1000）可以达到 100% 稳定率（之前的 baseline 分析）
- DPO 试图把这个 12% 提高到更高，但**自回归语言模型的生成多样性与其架构深度绑定**，微调很难改变生成分布的整体形状

## 6. 实验价值总结

### 正面成果
1. **排除了损失函数假设**: DPO/cDPO/SimPO 表现几乎相同，确认瓶颈不在优化算法
2. **排除了超参数假设**: β 从 0.1 提高到 2.5，LR 降低 5 倍，步数增加 4 倍，均无效
3. **100% Novelty**: 所有生成的稳定结构都是训练集中不存在的，证明模型有真正的创造力
4. **建立了完整的实验基础设施**: 自动化 pipeline、评估工具、对比报告生成

### 关键数据点（适合论文报告）
- DPO 在单 prompt 晶体生成上的提升 < 0.2%，不具统计显著性
- 空间群分布在 DPO 前后完全不变，说明模型权重几乎未被有效更新
- pi_delta 在训练末期仅为 0.05（接近随机），说明模型未能学到有效偏好

## 7. 可行的下一步方向

### 方向 A: 多 Prompt DPO（最优先）
- 使用不同 Z 值（Z=1,2,4,8）生成不同的 prompt
- 或使用不同元素组合（Li₃PS₄, NaCl, TiO₂ 等）的 prompt
- 每个 prompt 对应该材料的特定偏好对
- **预期**: 多 prompt 可以避免梯度冲突，让 DPO 信号更清晰

### 方向 B: Reward-Guided Decoding（无需训练）
- 保持 baseline 模型不变
- 在采样时用 MatGL 能量作为 reward signal 引导 token 选择
- 类似 RLHF 中的 Best-of-N 但更高效
- **预期**: 直接在推理时优化，绕过 DPO 训练的根本问题

### 方向 C: 结构级别的 DPO（而非 Token 级别）
- 不在 token 序列上做 DPO，而是在结构参数空间（空间群、晶胞参数）上做偏好优化
- 需要将 CIF 解析为结构化表示后再训练
- **预期**: 更直接的优化信号，但需要大幅修改架构

### 方向 D: 将当前负结果写成论文
- 标题方向："DPO for Crystal Structure Generation: Why Token-Level Preference Optimization Fails"
- 贡献：首次系统性地展示 DPO 在科学结构生成上的局限性
- 数据充分：4 组实验 × 50K 样本，有统计说服力
