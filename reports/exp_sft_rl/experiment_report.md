# DPO-CrystaLLM 实验报告：SFT + RL 两阶段多组分管线

> 实验名称: `exp_sft_rl`
> 完成时间: 2026-02-21 08:53 CST
> 实验周期: 2026-02-15 ~ 2026-02-21（约 6 天）
> 硬件: NVIDIA GeForce RTX 4090D (24GB), Ubuntu 22.04

---

## 1. 研究背景与动机

### 1.1 前期实验回顾

在此前的一系列实验中，我们尝试了多种方法提升 CrystaLLM 生成晶体结构的热力学稳定率（Ehull < 0.05 eV/atom），但效果均不理想：

| 实验 | 方法 | 目标体系 | 稳定率变化 | 结论 |
|------|------|---------|-----------|------|
| exp_final_50k | DPO | LiFePO4 | 12.12% → 12.27% (+0.15%) | 改善低于统计显著性 |
| exp_ablation_dpo | DPO (调参) | LiFePO4 | 12.12% → 12.16% (+0.04%) | 参数调整无效 |
| exp_ablation_cdpo | cDPO | LiFePO4 | 12.12% → 12.14% (+0.02%) | 无显著差异 |
| exp_ablation_simpo | SimPO | LiFePO4 | 12.12% → 12.14% (+0.02%) | 无显著差异 |
| exp_sft_stable | SFT | LiFePO4 | 12.12% → 11.03% (-1.1%) | 稳定率反降 |

### 1.2 根因分析

通过对基线生成结构的深入分析，发现 DPO 失效的三个核心原因：

1. **Reward 信号过弱**: 仅使用 MatGL 能量做二值判定（稳定/不稳定），导致偏好对质量差（pi_delta ≈ 0.05，接近随机）
2. **单一 Prompt 梯度冲突**: 所有 15,000 个偏好对共享同一 prompt（`data_Li4Fe4P4O16`），不同空间群样本的梯度方向互相矛盾
3. **空间群分布瓶颈**: 模型将 26.3% 生成能力用于 P2₁ 空间群（仅 0.4% 稳定率），而高稳定率的 Cmcm（39.8%）占比不足

### 1.3 导师建议

针对上述三个问题，导师提出三条改进建议：

1. **多维度 Reward 正则化**: 引入结构合理性、任务难度、组分匹配作为 reward 的正则项
2. **数据多样性**: 扩展到多组分体系，天然形成多 prompt 训练
3. **SFT + RL 两阶段流程**: 先用 SFT 偏移分布，再用 DPO 精细优化

本实验据此设计并实施了完整的 SFT + RL 两阶段多组分管线。

## 2. 实验设计

### 2.1 总体架构

```
Phase 1: 多组分基线生成 + MatGL 评分 + Ehull 估计
    → 每个组分 10,000 CIF
Phase 2: 多组分 SFT 训练
    → 从基线中筛选稳定结构，LoRA 微调
Phase 3: SFT 模型重新生成 + 复合 Reward 评分 + 偏好对构建
    → 用 SFT 模型为每个组分重新生成 10K CIF
Phase 4: Reward-Weighted DPO 训练
    → 在 SFT 模型上做多 prompt DPO
Phase 5: 最终评估
    → Baseline vs SFT vs SFT+DPO 三方对比
```

### 2.2 目标组分

| 组分 | 化学式 | Prompt (Z 扩展) | 晶体类型 | 选择理由 |
|------|--------|-----------------|---------|---------|
| LiFePO4 | Li₄Fe₄P₄O₁₆ | `data_Li4Fe4P4O16` | 橄榄石 | 主实验目标，已有充分基线数据 |
| NaCl | Na₄Cl₄ | `data_Na4Cl4` | 岩盐 | 简单二元体系，高对称性 |
| TiO2 | Ti₄O₈ | `data_Ti4O8` | 金红石/锐钛矿 | 中等复杂度二元体系 |
| BaTiO3 | BaTiO₃ | `data_BaTiO3` | 钙钛矿 | 三元体系，测试模型泛化能力 |

### 2.3 复合 Reward 函数

$$R_{\text{total}} = 0.4 \cdot R_{\text{energy}} + 0.2 \cdot R_{\text{structure}} + 0.2 \cdot R_{\text{difficulty}} + 0.2 \cdot R_{\text{composition}}$$

| 维度 | 权重 | 计算方法 | 物理含义 |
|------|------|---------|---------|
| R_energy | 0.4 | MatGL 能量 min-max 归一化到 [0,1] | 越低能量越好 |
| R_structure | 0.2 | 键长合理性 + 体积合理性 + 密度合理性 | 结构是否物理合理 |
| R_difficulty | 0.2 | 1 - 空间群经验稳定率 | 难空间群中生成稳定结构更应被奖励 |
| R_composition | 0.2 | 组分与目标的匹配度 (1.0/0.5/0.0) | 鼓励生成目标组分 |

### 2.4 超参数配置

**SFT 阶段 (Stage 1)**:

| 参数 | 值 | 说明 |
|------|-----|------|
| 训练步数 | 6,000 | ~2 epoch |
| 学习率 | 5×10⁻⁸ | 低学习率防灾难性遗忘 |
| 策略 | LoRA (rank=16) | 减少可训练参数 |
| 梯度累积 | 8 | 有效 batch size = 8 |
| Warmup | 200 步 | |
| Ehull 阈值 | 0.05 eV/atom | 筛选稳定结构 |

**DPO 阶段 (Stage 2)**:

| 参数 | 值 | 说明 |
|------|-----|------|
| 训练步数 | 4,000 | |
| 学习率 | 1×10⁻⁷ | |
| β | 2.5 | KL 惩罚系数 |
| 策略 | Full fine-tuning | 全参数训练 |
| 梯度累积 | 16 | 有效 batch size = 16 |
| Warmup | 200 步 | |
| Loss | Reward-weighted DPO | α = 1.0 |
| 偏好对 | 15,000 | 3 组分 × 5,000 |

**Reward-Weighted DPO Loss**:

$$\mathcal{L} = -\log \sigma\Big(\beta \cdot (f_\theta(y_w) - f_\theta(y_l)) + \alpha \cdot (r_w - r_l)\Big)$$

其中 $r_w, r_l$ 为偏好对中 chosen/rejected 样本的复合 reward。

### 2.5 采样多样性

生成阶段使用参数随机化增加结构多样性：
- Temperature: 在 [0.7, 1.3] 范围内随机采样
- Top-k: 在 [5, 20] 范围内随机采样

## 5. BaTiO3 失败分析

BaTiO3 是本实验中唯一完全失败的组分，需要专门分析。

### 5.1 问题表现

- 10,000 个生成结构中 **0 个** 匹配 BaTiO3 组分
- R_composition = 0.0000（所有样本）
- 配对构建阶段因 0 候选崩溃

### 5.2 生成分布

| 生成组分 | 数量 | 占比 |
|---------|------|------|
| BaO | 8,782 | 87.8% |
| BaO2 | 992 | 9.9% |
| Ba2O | 111 | 1.1% |
| 其他 (BaO3, Ba2O3, PuO 等) | 115 | 1.2% |
| **BaTiO3** | **0** | **0%** |

### 5.3 根因

CrystaLLM v1 small 使用 prompt `data_BaTiO3` 来条件生成。然而：

1. **Z=1 的 prompt 过短**: BaTiO3 配置 Z=1（`data_BaTiO3`），相比 Z=4 的其他组分（如 `data_Li4Fe4P4O16`），prompt 信息量不足
2. **三元化合物超出模型能力**: CrystaLLM small 可能在训练数据中缺乏足够的 ABO₃ 钙钛矿结构，导致无法生成正确的三元组分
3. **模型退化为 Ba-O 二元体系**: 模型"忽略"了 Ti 元素，退化为生成最简单的 Ba-O 体系

### 5.4 影响

- SFT 训练集中包含了 479 个 BaTiO3 稳定结构（来自 baseline 中 Ehull<0.05 的 BaO 等结构），这些结构虽然标记来源是 BaTiO3 目录，但实际组分为 Ba-O
- DPO 阶段跳过了 BaTiO3，实际只用了 3 个组分的偏好对

## 7. 结果讨论

### 7.1 SFT 阶段效果评估

**结论: SFT 阶段（LoRA, 6000 步, LR=5e-8）对模型几乎没有影响。**

- 三个组分的稳定率变化均 < 0.02%（统计噪声范围内）
- 能量分布几乎不变（均值差异 < 0.002 eV/atom）
- 复合 Reward 分数完全一致

**可能原因**:
1. **LoRA rank=16 过小**: 模型仅有 25.9M 参数，LoRA 可训练参数可能不足以影响生成分布
2. **学习率过低**: 5e-8 可能导致参数更新幅度极小
3. **SFT 数据量不足**: 2,830 个稳定结构相比模型原始训练数据量偏小
4. **基线与 SFT 数据高度重叠**: SFT 数据来自基线生成的稳定子集，模型已经"知道"如何生成这些结构

### 7.2 DPO 阶段效果评估

**结论: DPO 阶段产生了可检测但有限的效果，且方向不一致。**

正面效果:
- **TiO2 稳定率提升 +0.70%**: 从 512 → 582 个稳定结构，相对提升 13.7%
- **NaCl 稳定率提升 +0.15%**: 从 563 → 578 个稳定结构
- **NaCl 能量标准差大幅下降**: 从 3.41 → 0.76，模型学会了避免极端差的结构
- **LiFePO4 组分命中率提升 +5.4%**: 说明 DPO 确实影响了生成分布
- **极端值减少**: LiFePO4 最差能量从 154.8 降到 102.8, NaCl 从 331.1 降到 30.3

负面效果:
- **LiFePO4 稳定率下降 -0.37%**: 从 1,276 → 1,239 个稳定结构
- **TiO2 出现新的极端值**: 最差能量从 82.6 上升到 302.5

### 7.3 与前期实验对比

| 方法 | LiFePO4 Δ稳定率 | 核心改进 |
|------|-----------------|---------|
| DPO (exp_final_50k) | +0.15% | 单维 reward, 单组分 |
| DPO (exp_ablation_dpo) | +0.04% | 调参 β=2.5, 8000 步 |
| cDPO | +0.02% | Label smoothing |
| SimPO | +0.02% | Length-normalised |
| SFT (exp_sft_stable) | -1.10% | 只学好样本 |
| **SFT+DPO (本实验)** | **-0.37%** | 多维 reward + 多组分 + 两阶段 |

在 LiFePO4 上，本实验的 SFT+DPO 管线不仅没有改善，反而略有退步。但在 TiO2 和 NaCl 上表现优于之前的单组分实验。

### 7.4 为什么改善有限？

**1. SFT 阶段无效是最大问题**

两阶段方案的理论假设是: SFT 先偏移空间群分布 → DPO 再精细调参数。但实验中 SFT 完全没有偏移分布，DPO 仍然面临与之前相同的梯度冲突问题。

**2. 多组分带来的改善被稀释**

虽然引入了多 prompt（LiFePO4/NaCl/TiO2），但每个 prompt 的偏好对仍然在各自的空间群分布中冲突。多 prompt 解决了跨组分冲突，但同一组分内的冲突依然存在。

**3. Reward 信号仍然不够强**

复合 Reward 中 R_structure（≈0.999）和 R_difficulty（=0.88 固定）贡献几乎恒定，实际区分度主要来自 R_energy 和 R_composition。有效维度只有 2 个而非设计的 4 个。

**4. DPO 使用 Full Fine-tuning 可能过拟合**

配置中 SFT 使用 LoRA, 但 DPO 切换为 Full fine-tuning（25.9M 全参数），在 15,000 个偏好对上可能导致灾难性遗忘。

## 8. 关键发现总结

1. **LoRA SFT 在小模型上近乎无效**: CrystaLLM small (25.9M) 使用 LoRA rank=16, LR=5e-8 训练 6000 步后，生成分布几乎不变。说明 LoRA 在小模型上的微调能力严重受限，可能需要 Full fine-tuning 或更高 rank
2. **多组分 DPO 在部分体系上有效**: TiO2 稳定率提升 +0.70%（相对 +13.7%），NaCl +0.15%。多 prompt 训练确实改善了部分体系
3. **DPO 能显著降低极端值**: NaCl 能量标准差从 3.41 降到 0.76，最差值从 331 降到 30。模型学会了避免生成物理上荒谬的结构
4. **BaTiO3 揭示了 CrystaLLM 的组分生成局限性**: 模型无法生成三元化合物，退化为二元 Ba-O 体系
5. **复合 Reward 的实际区分度有限**: R_structure 和 R_difficulty 几乎恒定，未能提供有效正则化信号

## 9. 后续建议

### 短期改进

1. **SFT 阶段改为 Full fine-tuning**: 或提高 LoRA rank（如 64/128），并适当提高学习率（如 1e-6 ~ 5e-7），确保 SFT 真正偏移空间群分布
2. **同空间群内构建偏好对**: 在 Cmcm 内做低能量 vs 高能量配对，避免跨空间群梯度冲突
3. **动态 R_difficulty**: 按样本实际空间群计算难度系数，而非使用固定值 0.88

### 中期方向

4. **迭代式 SFT (ReST-EM)**: 每轮用新模型生成 → 筛选稳定 → 加入训练集 → 再训练，逐步放大稳定率
5. **更大 Z 值的 BaTiO3 prompt**: 尝试 `data_Ba4Ti4O12`（Z=4），提供更多结构信息

### 长期方向

6. **更大模型**: CrystaLLM small 的 25.9M 参数可能是根本瓶颈，考虑在更大模型上验证
7. **在线 RL (PPO/GRPO)**: 替代离线 DPO，允许模型实时探索和获得反馈
